# คู่มือการติดตั้ง Self-Hosted Supabase บนเซิร์ฟเวอร์ส่วนตัวอย่างละเอียด 🚀
(สำหรับธุรกิจระยะยาว 20-30 ปี ไร้ขีดจำกัด ฟรีค่าบริการซอฟต์แวร์ 100%)

คู่มือนี้จะอธิบายขั้นตอนการติดตั้ง **Supabase** ทั้งระบบ (ประกอบด้วย PostgreSQL, Authentication, Storage, และ API Gateway) ลงบนเซิร์ฟเวอร์ของตัวเอง ไม่ว่าจะเป็นเซิร์ฟเวอร์ระบบ **Linux (Ubuntu)** หรือ **Windows Server / Windows PC** โดยที่**โค้ด Next.js ของโปรเจกต์คุณจะใช้คำสั่งเดิมได้ทั้งหมด 100%** เพียงแค่เปลี่ยนที่อยู่ปลายทางเท่านั้น!

---

## 🏗️ ส่วนที่ 1: การเตรียมเครื่องเซิร์ฟเวอร์ระบบ Windows 🖥️
(สำหรับเครื่องเซิร์ฟเวอร์ระบบ Linux สามารถเลื่อนไปดูในส่วนที่ 2 ได้เลยครับ)

การรัน Supabase บนระบบ Windows Server หรือคอมพิวเตอร์ Windows ทั่วไป จะทำงานผ่านระบบ **Docker Desktop** ซึ่งติดตั้งง่ายและเป็นระบบภาพอินเตอร์เฟสใช้งานสะดวกครับ

### 1.1 ติดตั้งโปรแกรมที่จำเป็นบน Windows
1. **ติดตั้ง Docker Desktop สำหรับ Windows:**
   * เข้าไปดาวน์โหลดตัวติดตั้งฟรีที่เว็บอย่างเป็นทางการ 👉 **[Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)**
   * ดับเบิลคลิกติดตั้งตามขั้นตอน (แนะนำให้เลือกติ๊กถูกติดตั้งระบบ **WSL 2** ที่ตัวติดตั้งแนะนำ)
   * เมื่อติดตั้งเสร็จ ให้รีสตาร์ตเครื่องคอมพิวเตอร์ 1 ครั้ง และเปิดโปรแกรม Docker Desktop ทิ้งไว้ให้เป็นสีเขียว (Engine Running)
2. **ติดตั้ง Git (เลือกทำหรือไม่ทำก็ได้):**
   * หากต้องการดาวน์โหลดโค้ดผ่านคีย์บอร์ดให้ลง Git
   * **ทางเลือกที่ง่ายกว่า:** หากไม่มี Git ให้เปิดเบราว์เซอร์บนเซิร์ฟเวอร์แล้วกดดาวน์โหลดไฟล์ ZIP โครงสร้าง Supabase ตรงๆ จากลิงก์นี้แทนได้เลยครับ 👉 **[Download Supabase ZIP](https://github.com/supabase/supabase/archive/refs/heads/master.zip)**

## 📥 ส่วนที่ 2: ดาวน์โหลดและตั้งค่าระบบ Supabase (Step-by-Step)

ขั้นตอนนี้สามารถทำได้เหมือนกันทั้งบน Windows และ Linux ครับ:

### 2.1 ดึงโค้ดการติดตั้ง
* **สำหรับ Windows (แบบดาวน์โหลด ZIP):** 
  * แตกไฟล์ ZIP ที่โหลดมา (เช่น `supabase-master.zip`) ไปวางไว้ที่โฟลเดอร์ใดโฟลเดอร์หนึ่ง เช่น `C:\supabase`
  * เข้าไปยังโฟลเดอร์ย่อยย่อย: `C:\supabase\docker`
  * ทำการคัดลอกไฟล์ชื่อ `.env.example` แล้วเปลี่ยนชื่อไฟล์ที่คัดลอกใหม่เป็น **`.env`** (เปิดไฟล์แก้ไขด้วยโปรแกรม Notepad ทั่วไปได้เลย)

* **สำหรับ Linux (ผ่าน Terminal):**
  ```bash
  git clone --depth 1 https://github.com/supabase/supabase
  cd supabase/docker
  cp .env.example .env
  ```

### 2.2 ตั้งค่าความปลอดภัยและคีย์ระบบ (สำคัญที่สุด!)
เปิดไฟล์ `.env` บนโปรแกรมแก้ไขข้อความ (Notepad) และแก้ไขค่าความปลอดภัยเพื่อป้องกันการแฮกข้อมูล:

> [!WARNING]
> ห้ามใช้รหัสผ่านและคีย์ที่เป็นค่าเริ่มต้นจากระบบเด็ดขาด เพราะฐานข้อมูลของคุณจะถูกแฮกได้ทันทีหากเปิดใช้งานสู่สาธารณะ

คุณจำเป็นต้องสร้างรหัสผ่านและ JWT Token ของคุณเอง โดยใช้ค่าสุ่มความปลอดภัยสูง (สเปกคีย์สามารถพิมพ์สุ่มยาวๆ เองได้เลย หรือพิมพ์ผ่าน PowerShell ด้วยคำสั่งสุ่มดังนี้):
```powershell
# รันคำสั่งนี้ใน PowerShell เพื่อสุ่มรหัสความยาวสูง
[Guid]::NewGuid().ToString()
```
นำคีย์ที่สุ่มได้ไปใส่ในไฟล์ `.env` แทนค่าเดิมดังนี้:

1. **`POSTGRES_PASSWORD`:** เปลี่ยนเป็นรหัสผ่านฐานข้อมูลที่มีความปลอดภัยสูงของคุณเอง
2. **`JWT_SECRET`:** เปลี่ยนเป็นคีย์ลับความปลอดภัยสูงสำหรับระบบ Auth
3. **`ANON_KEY`:** (คีย์สำหรับดึงข้อมูลสาธารณะหน้าบ้าน)
4. **`SERVICE_ROLE_KEY`:** (คีย์สิทธิ์สูงสุดสำหรับหลังบ้าน)

#### 🔑 วิธีการสร้าง ANON_KEY และ SERVICE_ROLE_KEY ใหม่:
เพื่อให้คีย์ทั้งสองทำงานร่วมกับ `JWT_SECRET` ใหม่ได้อย่างถูกต้อง แนะนำให้เข้าไปสร้างที่เว็บดึงคีย์ของ Supabase โดยตรงที่ 👉 **[Supabase JWT Generator](https://jwt.io/)** (หรือสามารถใช้ค่ารหัสสุ่มความยาวสูงพิเศษใส่แทนก็ได้เช่นกันหากเป็นการใช้งานส่วนตัว)

## 🚀 ส่วนที่ 3: สั่งรันระบบ Supabase ขึ้นทำงาน

### 3.1 วิธีสั่งรันบน Windows (ผ่าน PowerShell / Command Prompt)
1. กดปุ่ม `Start` > พิมพ์ค้นหา **PowerShell** แล้วคลิกเปิดขึ้นมา
2. พิมพ์คำสั่งเพื่อเคลื่อนย้ายไปยังโฟลเดอร์ติดตั้ง (เปลี่ยนเส้นทางตามโฟลเดอร์ของคุณ):
   ```powershell
   cd C:\supabase\docker
   ```
3. สั่งงานระบบ Docker ให้ดาวน์โหลดและเปิดทำงาน:
   ```powershell
   docker compose pull
   docker compose up -d
   ```
   *(คุณจะเห็นแถบดาวน์โหลดในโปรแกรม Docker Desktop และระบบทุกอย่างจะเริ่มเปิดทำงานโดยอัตโนมัติ สังเกตสถานะไฟจะเป็นสีเขียวทั้งหมดในโปรแกรม Docker Desktop)*

### 3.2 การเปิดหน้าแดชบอร์ดหลังบ้าน (Supabase Studio)
เมื่อระบบรันเสร็จแล้ว คุณสามารถเข้าจัดการตารางข้อมูลทั้งหมดได้ทันทีผ่านเบราว์เซอร์:
* เปิดเบราว์เซอร์ไปที่: `http://localhost:8000` (หรือ `http://ไอพีของเครื่องเซิร์ฟเวอร์:8000`)
* เข้าสู่ระบบด้วยรหัสผ่านที่คุณตั้งไว้ในค่า `STUDIO_PASSWORD` ในไฟล์ `.env`
* พอร์ต API หน้าบ้านคือ `http://localhost:8000` และพอร์ตสำหรับต่อฐานข้อมูลตรงคือ `5432`

## 🔄 ส่วนที่ 4: การนำโครงสร้างเดิมไปใช้และการเชื่อมโยงโปรเจกต์ Next.js

### 4.1 นำโครงสร้างตารางเดิมจากไฟล์ `live_schema.sql` ไปสร้างในระบบใหม่
1. เปิดเบราว์เซอร์ไปที่ **Supabase Studio** เครื่องใหม่ของคุณ (`http://localhost:8000`)
2. ไปที่เมนู **SQL Editor** (รูปสัญลักษณ์ `SQL` ด้านซ้ายมือ) > คลิก **New query**
3. เปิดไฟล์ **`live_schema.sql`** ที่เราก๊อปปี้มา เปิดและคัดลอกโค้ดทั้งหมด ไปวางในช่อง SQL Editor แล้วกด **Run**
4. ตารางทั้งหมด, RPC, RLS, และ Indexes จะถูกย้ายมาระบบใหม่ 100% ทันที!

### 4.2 การอัปเดตไฟล์ `.env.local` ในโปรเจกต์ Next.js ของคุณ
ในโปรเจกต์ Next.js เครื่องใหม่ของคุณ ให้เปลี่ยนค่าแปรสภาพแวดล้อมดังนี้:

```env
# เปลี่ยนให้ชี้ไปที่เซิร์ฟเวอร์ Windows เครื่องใหม่ของคุณ
NEXT_PUBLIC_SUPABASE_URL=http://ไอพีเครื่องเซิร์ฟเวอร์ของคุณ:8000
NEXT_PUBLIC_SUPABASE_ANON_KEY=ANON_KEY_ตัวใหม่ที่คุณสร้างในไฟล์_.env_ของเซิร์ฟเวอร์
SUPABASE_SERVICE_ROLE_KEY=SERVICE_ROLE_KEY_ตัวใหม่ที่คุณสร้างในไฟล์_.env_ของเซิร์ฟเวอร์
SUPABASE_PROJECT_ID=lcadzsnvxunmtkdhhrty # ใส่ ID อะไรก็ได้
```

## 🛡️ ส่วนที่ 5: ระบบสำรองข้อมูลอัตโนมัติบน Windows (Task Scheduler)
> [!IMPORTANT]
> เมื่อคุณเปลี่ยนมาดูแลระบบฐานข้อมูลเอง การปกป้องข้อมูลเป็นหน้าที่ของคุณ 100% หากฮาร์ดดิสก์เสียหรือเครื่องดับ ข้อมูลทางธุรกิจทั้งหมดอาจสูญหายได้ทันที

บน Windows เราจะใช้โปรแกรมตั้งค่าตารางงาน **Windows Task Scheduler** ร่วมกับสคริปต์ Batch File (`.bat`) เพื่อสำรองข้อมูลโดยอัตโนมัติทุกวันตอนเที่ยงคืนครับ:

### 5.1 สร้างสคริปต์สำรองข้อมูลสำหรับ Windows
1. สร้างโฟลเดอร์สำหรับเก็บไฟล์สำรอง เช่น `C:\db_backups`
2. คลิกขวาที่หน้าจอว่างๆ > เลือก **New > Text Document**
3. เปิดไฟล์ข้อความนั้นขึ้นมา แล้วใส่คำสั่งสคริปต์ก๊อปปี้ฐานข้อมูลดังนี้:

```batch
@echo off
:: โฟลเดอร์ที่ต้องการเก็บไฟล์สำรอง
set BACKUP_DIR=C:\db_backups
:: ตั้งชื่อไฟล์ด้วย วันที่ และ เวลา
set TIMESTAMP=%date:~10,4%%date:~4,2%%date:~7,2%_%time:~0,2%%time:~3,2%%time:~5,2%
set TIMESTAMP=%TIMESTAMP: =0%
set FILENAME=tynoodle_backup_%TIMESTAMP%.sql

echo Starting database backup...

:: สั่งให้ Docker ทำการ Dump ข้อมูล PostgreSQL ออกมาเป็นไฟล์ในเครื่อง Windows
docker exec -t supabase-db pg_dump -U postgres -d postgres -F p > "%BACKUP_DIR%\%FILENAME%"

echo Backup completed successfully: %FILENAME%
```

4. กดบันทึกไฟล์ (Save As) โดยเปลี่ยนนามสกุลไฟล์จาก `.txt` เป็น **`.bat`** (เช่น ตั้งชื่อว่า **`backup_database.bat`**) และเลือก Save as type เป็น *All Files*

### 5.2 ตั้งเวลาให้รันทุกคืนอัตโนมัติ (Windows Task Scheduler)
1. กดปุ่ม `Start` บน Windows > พิมพ์ค้นหาคำว่า **Task Scheduler** แล้วคลิกเปิดขึ้นมา
2. ที่แถบด้านขวา คลิกเลือก **Create Basic Task...**
3. **Name:** ตั้งชื่อว่า `Supabase_Daily_Backup` > คลิก Next
4. **Trigger:** เลือกเป็น **Daily** (ทุกวัน) > คลิก Next
5. **Start:** ตั้งค่าเวลาเป็น **00:00:00** (เที่ยงคืนตรง) และคลิก Next
6. **Action:** เลือก **Start a program** > คลิก Next
7. **Program/script:** คลิกปุ่ม **Browse...** และเลือกไฟล์ **`backup_database.bat`** ที่เราสร้างไว้ในหัวข้อ 5.1 > คลิก Next
8. คลิก **Finish** เป็นอันเสร็จสมบูรณ์!
