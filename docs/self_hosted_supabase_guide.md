# คู่มือการติดตั้ง Self-Hosted Supabase บนเซิร์ฟเวอร์ส่วนตัวอย่างละเอียด 🚀
(สำหรับธุรกิจระยะยาว 20-30 ปี ไร้ขีดจำกัด ฟรีค่าบริการซอฟต์แวร์ 100%)

คู่มือนี้จะอธิบายขั้นตอนการติดตั้ง **Supabase** ทั้งระบบ (ประกอบด้วย PostgreSQL, Authentication, Storage, และ API Gateway) ลงบนเซิร์ฟเวอร์ของตัวเอง ไม่ว่าจะเป็นเครื่องคอมพิวเตอร์ที่ออฟฟิศ หรือการเช่า Cloud VPS (เช่น DigitalOcean, Hetzner, หรือ AWS EC2) โดยที่**โค้ด Next.js ของโปรเจกต์คุณจะใช้คำสั่งเดิมได้ทั้งหมด 100%** เพียงแค่เปลี่ยนที่อยู่ปลายทางเท่านั้น!

---

## 🏗️ ส่วนที่ 1: การเตรียมเครื่องเซิร์ฟเวอร์ (Prerequisites)

ก่อนเริ่มติดตั้ง คุณต้องมีเครื่องเซิร์ฟเวอร์ปลายทาง โดยสามารถเลือกได้ 2 แบบ:
1. **เครื่องเซิร์ฟเวอร์ที่ออฟฟิศ (Local PC):** คอมพิวเตอร์สเปกปานกลางขึ้นไป (แนะนำลง OS เป็น **Ubuntu Server 22.04 / 24.04 LTS** เพื่อความเสถียร หรือจะใช้ Windows + Docker Desktop ก็ได้เช่นกัน)
2. **Cloud VPS:** แนะนำเช่าเครื่องจากผู้ให้บริการ เช่น DigitalOcean, Hetzner, หรือ Linode โดยเลือก OS เป็น **Ubuntu 24.04 LTS** (สเปกขั้นต่ำ: RAM 2GB, CPU 2 Cores) ราคาจะอยู่ที่ประมาณ 180 - 350 บาทต่อเดือน

### 1.1 ติดตั้ง Docker และ Docker Compose บนเซิร์ฟเวอร์
หากใช้ **Ubuntu Linux** ให้รันคำสั่งเหล่านี้เพื่อติดตั้ง Docker:
```bash
# อัปเดตระบบ
sudo apt update && sudo apt upgrade -y

# ติดตั้ง Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# ตรวจสอบการติดตั้ง Docker
docker --version
docker compose version
```

## 📥 ส่วนที่ 2: ดาวน์โหลดและตั้งค่าระบบ Supabase (Step-by-Step)

Supabase มีเทมเพลตสำหรับเริ่มรันผ่าน Docker Compose เตรียมไว้ให้เราแล้ว ให้ทำการรีโมตเข้าไปที่เซิร์ฟเวอร์ของคุณและทำตามขั้นตอนดังนี้ครับ:

### 2.1 ดึงโค้ดการติดตั้งจาก Supabase
```bash
# Clone เฉพาะโฟลเดอร์สำหรับรัน Docker
git clone --depth 1 https://github.com/supabase/supabase

# เข้าไปยังโฟลเดอร์รันระบบ
cd supabase/docker

# ก๊อปปี้ไฟล์ตั้งค่าตัวอย่าง
cp .env.example .env
```

### 2.2 ตั้งค่าความปลอดภัยและคีย์ระบบ (สำคัญที่สุด!)
เปิดไฟล์ `.env` ขึ้นมาแก้ไขค่าความปลอดภัย (บน Linux สามารถใช้โปรแกรม `nano .env` ในการแก้ไข):

> [!WARNING]
> ห้ามใช้รหัสผ่านและคีย์ที่เป็นค่าเริ่มต้นจากระบบเด็ดขาด เพราะฐานข้อมูลของคุณจะถูกแฮกได้ทันทีหากเปิดใช้งานสู่สาธารณะ

คุณจำเป็นต้องสร้างรหัสผ่านและ JWT Token ของคุณเอง โดยใช้คำสั่งสร้างสุ่มบน Terminal ของคุณเอง:
```bash
# คำสั่งสร้างรหัสสุ่ม 32 ตัวอักษร
openssl rand -hex 32
```
รันคำสั่งนี้หลายๆ ครั้งแล้วนำค่าที่ได้ไปแทนที่ในไฟล์ `.env` ดังนี้:

1. **`POSTGRES_PASSWORD`:** เปลี่ยนเป็นรหัสผ่านฐานข้อมูลที่มีความปลอดภัยสูงของคุณเอง
2. **`JWT_SECRET`:** เปลี่ยนเป็นคีย์ลับความปลอดภัยสูงสำหรับระบบ Auth
3. **`ANON_KEY`:** (คีย์สำหรับดึงข้อมูลสาธารณะหน้าบ้าน)
4. **`SERVICE_ROLE_KEY`:** (คีย์สิทธิ์สูงสุดสำหรับหลังบ้าน)

#### 🔑 วิธีการสร้าง ANON_KEY และ SERVICE_ROLE_KEY ใหม่:
เพื่อให้คีย์ทั้งสองทำงานร่วมกับ `JWT_SECRET` ใหม่ได้อย่างถูกต้อง ให้ใช้เว็บไซต์สร้าง JWT ของระบบ Supabase โดยตรงที่ 👉 **[Supabase JWT Generator](https://jwt.io/)** หรือรันผ่านสคริปต์สั้นๆ เพื่อสร้างโค้ดตามความต้องการ (หรือใช้ค่าสุ่มความยาวสูงแบบที่โปรเจกต์แจกให้ก็ได้)

## 🚀 ส่วนที่ 3: สั่งรันระบบ Supabase ขึ้นทำงาน

เมื่อตั้งค่าไฟล์ `.env` เรียบร้อยแล้ว ให้ทำการดาวน์โหลดอิมเมจและสั่งเปิดเซิร์ฟเวอร์ผ่าน Docker:

```bash
# ดาวน์โหลด Docker Images ทั้งหมด
docker compose pull

# สั่งรัน Supabase ทั้งหมดทำงานในโหมดเบื้องหลัง (Background)
docker compose up -d
```

ตรวจสอบความเรียบร้อยว่าระบบทุกส่วนเปิดทำงานสำเร็จ:
```bash
docker compose ps
```
คุณจะพบรายชื่อเซิร์ฟเวอร์ระบบทั้งหมดของ Supabase รันขึ้นมาอย่างสมบูรณ์แบบครับ! 

* **Supabase Studio (หน้าบอร์ดจัดการฐานข้อมูลเหมือนบนเว็บจริง):** จะสามารถเข้าใช้งานได้ผ่านเบราว์เซอร์ที่พอร์ต `8000` (เช่น `http://ไอพีเซิร์ฟเวอร์ของคุณ:8000` โดยรหัสผ่านเริ่มต้นจะตั้งอยู่ในค่า `STUDIO_PASSWORD` ในไฟล์ `.env`)
* **API URL:** จะเข้าถึงผ่าน `http://ไอพีเซิร์ฟเวอร์ของคุณ:8000` 
* **Database Connection Port:** พอร์ตตรงคือ `5432`

## 🔄 ส่วนที่ 4: การนำโครงสร้างเดิมไปใช้และการเชื่อมโยงโปรเจกต์ Next.js

### 4.1 นำโครงสร้างตารางเดิมจากไฟล์ `live_schema.sql` ไปสร้างในระบบใหม่
1. เปิดเบราว์เซอร์ไปที่ **Supabase Studio** เครื่องใหม่ของคุณ (`http://ไอพีเซิร์ฟเวอร์ของคุณ:8000`)
2. ล็อกอินด้วยบัญชีที่ตั้งไว้ในไฟล์ `.env`
3. ไปที่เมนู **SQL Editor** (รูปสัญลักษณ์ `SQL` ด้านซ้ายมือ) > คลิก **New query**
4. เปิดไฟล์ **`live_schema.sql`** ที่เรา Dump ได้ในเครื่องของคุณ ก๊อปปี้ข้อความทั้งหมดไปวางในช่อง แล้วกด **Run**
5. ตารางทั้งหมด, RPC, RLS, และ Indexes จะถูกโคลนไปยังระบบใหม่ 100% ทันที!

### 4.2 การอัปเดตไฟล์ `.env.local` ในโปรเจกต์ Next.js ของคุณ
ในโปรเจกต์ Next.js ในเครื่องใหม่ของคุณ ให้เปลี่ยนค่าแปรสภาพแวดล้อมดังนี้:

```env
# เปลี่ยนให้ชี้ไปที่เซิร์ฟเวอร์ของคุณเอง
NEXT_PUBLIC_SUPABASE_URL=http://ไอพีเซิร์ฟเวอร์ของคุณ:8000
NEXT_PUBLIC_SUPABASE_ANON_KEY=ANON_KEY_ตัวใหม่ที่คุณสร้างในไฟล์_.env_ของเซิร์ฟเวอร์
SUPABASE_SERVICE_ROLE_KEY=SERVICE_ROLE_KEY_ตัวใหม่ที่คุณสร้างในไฟล์_.env_ของเซิร์ฟเวอร์
SUPABASE_PROJECT_ID=lcadzsnvxunmtkdhhrty # ใส่ ID เดิมหรือ ID อะไรก็ได้
```

เพียงเท่านี้ ระบบ Next.js ของโปรเจกต์ใหม่ของคุณก็จะเชื่อมต่อกับฐานข้อมูลในเซิร์ฟเวอร์ส่วนตัวของคุณโดยตรงทันที **และรันไปได้ยาวๆ 20-30 ปีโดยไม่มีการจำกัดขนาดข้อมูล 500MB อีกต่อไป!**

## 🛡️ ส่วนที่ 5: ระบบสำรองข้อมูลอัตโนมัติ (Automated Backup)
> [!IMPORTANT]
> เมื่อคุณเปลี่ยนมาดูแลระบบฐานข้อมูลเอง การปกป้องข้อมูลเป็นหน้าที่ของคุณ 100% หากฮาร์ดดิสก์เสียหรือเครื่องดับ ข้อมูลทางธุรกิจทั้งหมดอาจสูญหายได้ทันที

นี่คือวิธีการทำ **ระบบสำรองข้อมูลอัตโนมัติทุกวันตอนเที่ยงคืน (Daily Cron Job Backup)** เพื่อเซฟเก็บไฟล์ไว้ในเครื่องอื่นหรือ Cloud Storage ปลอดภัย 100% ครับ:

### 5.1 สร้างสคริปต์สำรองข้อมูล
สร้างไฟล์สคริปต์ชื่อ `backup.sh` ไว้บนเซิร์ฟเวอร์ของคุณ:
```bash
nano /home/ubuntu/backup.sh
```

เขียนโค้ดเหล่านี้ลงในสคริปต์:
```bash
#!/bin/bash

# ที่อยู่จัดเก็บไฟล์สำรอง
BACKUP_DIR="/home/ubuntu/db_backups"
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
FILENAME="tynoodle_backup_${TIMESTAMP}.sql"

# รหัสผ่านและที่อยู่ฐานข้อมูล
DB_URL="postgresql://postgres:รหัสผ่านฐานข้อมูลใหม่@localhost:5432/postgres"

# สร้างโฟลเดอร์เก็บไฟล์หากยังไม่มี
mkdir -p ${BACKUP_DIR}

# ทำการสำรองข้อมูลโครงสร้าง + ข้อมูลจริงทั้งหมด
pg_dump ${DB_URL} -f ${BACKUP_DIR}/${FILENAME}

# (ออปชันเสริม) ลบไฟล์เก่าทิ้งหลังจากเก็บไว้แล้ว 30 วัน เพื่อไม่ให้หนักเครื่อง
find ${BACKUP_DIR} -type f -mtime +30 -name "*.sql" -exec rm {} \;

echo "Database backup completed: ${FILENAME}"
```

มอบสิทธิ์ในการรันสคริปต์ให้ระบบ:
```bash
chmod +x /home/ubuntu/backup.sh
```

### 5.2 ตั้งค่าให้สคริปต์ทำงานอัตโนมัติทุกคืนด้วย Cron Job
เปิดระบบ Cron Job ของเครื่องเซิร์ฟเวอร์:
```bash
crontab -e
```

พิมพ์บรรทัดนี้ต่อท้ายสุดของไฟล์ (สคริปต์จะสั่งทำงานอัตโนมัติ **ทุกคืน ณ เวลา 00:00 น.** เป๊ะๆ):
```text
0 0 * * * /home/ubuntu/backup.sh >> /home/ubuntu/backup.log 2>&1
```
